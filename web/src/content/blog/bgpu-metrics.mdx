### 2. Database to Astro MDX Synchronization
We implemented a live synchronization service [blog_mdx.py](file:///home/farma/programming/projects/duckops/api/src/app/services/blog_mdx.py) and hooked it into the FastAPI post endpoints in [__init__.py](file:///home/farma/programming/projects/duckops/api/src/app/__init__.py):
* **Real-time Synchronization on CRUD**: 
  * On **Create** (`POST /posts`), an `.mdx` file is instantly generated inside `web/src/content/blog/` using the slug as the filename.
  * On **Update** (`PUT /posts/{id}`), the MDX file content, title, and components are re-generated. As requested, updates do *not* delete older slug files from disk to prevent losing historic static assets.
  * On **Delete** (`DELETE /posts/{id}`), the MDX file corresponding to the active slug is deleted from disk to keep the static content repository clean.
* **Component Injection**: The generator automatically wires and writes `<PostApiData>` with the post UUID, and imports/embeds the `<MetricsChart slug={slug} client:load />` component if `data_path` is specified in the request.
* **Standard Placeholders**: Frontmatter attributes like `description` and `heroImage` are populated with generic placeholders for robust Astro generation.
